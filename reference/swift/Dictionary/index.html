Swift Dictionary

What is a Swift Dictionary and how to create?

Tested on: Xcode 12.2, Swift 5.3

Unlike Array which is ordered collection, dictionary is another collection but is it unordered. Dictionary contains a keys and value. The key cannot be appear more than once but different key can have the same value. These keys help you to identify which value it is related to. As shown in the image below:

The image below shown a key which is the name of the person and the key that linked to it is the values which is their age. For example, from here, we known that Tylor is 28 years old and Wes is 24.

When should you use dictionary?

Dictionary is useful when you want to get the value by it identifier or name. The main different between array and dictionary is that array fetch a value by it index which is sequential while dictionary fetch by it keys and the keys doesn't have any order in it.

How to create a Dictionary?

To create an dictionary in Swift, you use square brackets and inside the square brackets it contains key-value paris and is separated by commas. As you know Swift will infer the data type and the example below is [String: Int]

Swift Code :

let ages = ["Wes": 24, "Evan": 20, "Jes": 5, "Tylor": 28]

To create an empty dictionary, you can required to state the data type explicitly.

Swift Code :

let ages: [String:Int] = [:]

How to access the values?

Accessing the value in dictionary is similar to Array using subscript or square brackets. However, you don't access the value by its index but rather you call its keys. For example, you want to get Wes' age and you can type:

Swift Code :

let ages = ["Wes": 24, "Evan": 20, "Jes": 5, "Tylor": 28]
print(ages["Wes"])

Output :

24

How to modify an array? (Adding, Removing and Updating)

Updating and Adding Values

You can change the value by using subscripting or .updateValue(Value, forKey: Key). If the key doesn't exist, this method will add a new pair of value in.

For example, Wes's age is enter incorrectly and you can change his age by typing this (below) if the keys exist the value will change. (The example below uses subscripting as an example). This method can also be used to create a new key and value pair.

Swift Code :

var ages = ["Wes": 24, "Evan": 20, "Jes": 5, "Tylor": 28]
ages["Wes"] = 20
print(ages)

Output :

["Wes": 20, "Evan": 20, "Jes": 5, "Tylor": 28]

For example, Chua is 22 year old and you can add him by typing this (below) if the key does not exist, new key will be created. (The example below uses .updateValue as an example). This method can also used to update a value.

As you can see, chua is added anywhere in the dictionary and not the last one. This is becuase Dictionary is an unordered collection.

Swift Code :

var ages = ["Wes": 24, "Evan": 20, "Jes": 5, "Tylor": 28]
ages.updateValue(22, forKey: "chua")
print(ages)

Output :

["Evan": 20, "chua": 22, "Wes": 24, "Tylor": 28, "Jes": 5]

Removing a Key and Value Pair

You can remove a value by using .subscripting and assign a value of .nil to it. Moreover, you can use .removeValue(forKey: Key) to remove a pair.

Swift Code :

var ages = ["Wes": 24, "Evan": 20, "Jes": 5, "Tylor": 28]
ages["Evan"] = nil
print(ages)

Output :

["Wes": 24, "Jes": 5, "Tylor": 28]


Swift Code :

var ages = ["Wes": 24, "Evan": 20, "Jes": 5, "Tylor": 28]
ages.removeValue(forKey: "Evan")
print(ages)

Output :

["Wes": 24, "Jes": 5, "Tylor": 28]

Array methods and properties

.count

.count is a dictionary's property and it is used to count the number of key-value paris in the dictionary.

Swift Code :

var ages = ["Wes": 24, "Jes": 5, "Tylor": 28]
print(ages.count)

Output :

3

.isEmpty

.isEmpty is a dictionary's property and it is used to check whether is the dictionary is empty.

Swift Code :

var ages = ["Wes": 24, "Jes": 5, "Tylor": 28]
print(ages.isEmpty)

Output :

false

.reserveCapacity(Int)

.reserveCapacity can be use to increase the performance if you know how much the data, the dictionary will be storing. .reserveCapacity(Int) take in a parameter which is a Int and it indicated the size of the dictionary.

The reason to add this is becuase newly allocated storage might be greater than the requested capacity if Swift determine the size by itself. By stating it, you tell Swift the size of the minimum capacity and Swift will not allocate more space for it. Therefore, avoiding over allocation.

Swift Code :

var ages = ["Wes": 24, "Jes": 5, "Tylor": 28]
ages.reserveCapacity(5)

Output :

false
